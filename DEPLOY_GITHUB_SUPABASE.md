# Публикация на GitHub Pages + Supabase

## 1. Создайте Supabase-проект

1. Зарегистрируйтесь в Supabase.
2. Создайте новый проект.
3. Откройте SQL Editor и выполните содержимое файла `supabase-schema.sql`.
4. Скопируйте `Project URL` и `anon public key` из настроек проекта.

Если таблица уже создана раньше, просто выполните обновлённый `supabase-schema.sql` ещё раз, чтобы добавить политики для редактирования и удаления.

## 2. Подключите сайт к Supabase

1. Откройте `recipes.html`.
2. Найдите в начале скрипта поля:
   - `SUPABASE_URL`
   - `SUPABASE_ANON_KEY`
3. Вставьте туда свои значения.

## 3. Залейте файлы в GitHub

Загрузите в репозиторий:
- `index.html`
- `recipes.html`
- `supabase-schema.sql`
- `DEPLOY_GITHUB_SUPABASE.md`

## 4. Включите GitHub Pages

1. Откройте Settings репозитория.
2. Перейдите в Pages.
3. Выберите публикацию из `main` branch.
4. Сохраните настройки и дождитесь ссылки.

## 5. Как это будет работать

- Пользователи открывают GitHub Pages-сайт.
- Сайт читает рецепты из Supabase.
- Вы вводите пароль `123` и добавляете новые рецепты.
- Все посетители сразу видят общий фид.
